Using Password Vault

Introduction

Password Vault allow you to securely save all the usernames and passwords you use online in one place.

You only need to remember one thing - your encryption key. This can be a number, a word, a phrase but you should memorise it and never tell or record it anywhere.

This key securely encrypts all your other usernames and passwords, meaning you only have to remember that one key, not the hundreds of passwords you use day to day.

It is very important to never forget that key. It is not stored anywhere in the app and if you do forget it, you will NOT be able to retrieve your saved usernames and passwords.

How to use Password Vault

All usernames and passwords are related to a 'site'. To start, you need to add a site in the manager page by clicking the 'Add Site' button. This will take you to screen which allows you to name the site, add the username and password you use to access that site, and choose your encryption key. All input fields are limited to a maximum of 50 characters for security reasons.

You save the record by clicking the 'Add Site' button. Once a site is saved you'll see it listed under 'Site List' on the manager page.

You can click on that site list entry to view the details you saved for that site. This takes you to a screen where you can enter your encryption key to decrypt the username and password you saved for that site. This screen also allows you to delete a site, change the saved username or password and also change your encryption key.

And that's it! Remember your encryption key and all your other usernames and passwords can be stored securely without having to commit them to memory or scribble them down on pieces of paper!

FAQs

  • You say I only have to remember one key, but can I use a different key to store different site details?
  • Yes you can, if you choose, use a different key for different site records. It's up to you.
  • But, I also have to remember my username and password for Password Vault, as well as my key...
  • True, true, but if you do forget either your username or password for Password Vault, you can recover them here. So the only thing you absolutely have to remember is your encryption key.
  • Can I change my encryption key?
  • Yes, to do so log in to Password Vault, view a site that you've already saved and decrypt it with its current key (the key that you want to change). You can then amend the key by editing it in the input field and then re-encrypt the site record using the new key.
  • Is the encryption key ever stored on my computer or sent over the internet?
  • Absolutely not, the encryption key is held in memory only so long as its value is in the encryption key field. It's never stored in a database, a file or any other data structure and it's never transmitted to the server.
  • So what is sent over the internet?
  • The only un-encrypted data sent over the internet is your email address, username and the name you give to your sites in the site list. However your email address and username are both securely encrypted before they are stored in the database. The only data stored in plaintext in the database is the name of your site record.
  • How is the data encrypted?

  • There are two encryption methods used in the app.

    Your email address and password for Password Vault are hashed using the SHA256 algorithm. The SHA hash functions were designed by the National Security Agency (NSA). SHA-1 is the most established of the existing SHA hash functions, and it's used in a variety of security applications and protocols. SHA-256 used in this app, isn't as widely used as SHA-1, but it appears to provide much better security.

    Your username for Password Vault and the usernames and passwords associated to your site records are encrypted using AES.

    The Advanced Encryption Standard (AES) is a U.S. Federal Information Processing Standard (FIPS). It was selected after a 5-year process where 15 competing designs were evaluated. This app supports AES-128, AES-192, and AES-256. It will pick the variant by the size of the encryption key you choose. If you use a passphrase, then it will generate a 256-bit key.

    All encryption for this app is handled using the excellent CryptoJS library, with full credit for the software and explanation of the hashing and encrypting algorithms going to the author, Brix.

  • OK, you told me not to forget my encryption key, but I have. What can I do?
  • Ummm...unfortunately you will not be able to retrieve details for any site encrypted with that key. Your only option is to contact the actual site in question and request a new username and / or password. You can delete the site entry in Password Vault and when you have new username and password details you can create a new site record in Password Vault with your new log-on credentials. We cannot recover your encryption key, ever!
